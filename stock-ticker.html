Project Experience

For this project, I had to design and code a page that returned information from a stock ticker symbol that a user entered. I was free to use any API and could display any information. I ended up using Alpha Vantage due to its simplicity and used their Time Series Stock API for Quote Endpoint data. The main caveat of this project was that its structure had to adhere to the MVC (Model, View, Controller) pattern. Much of my knowledge of web applications at that point used ASP.Net and Entity Framework, which is structured differently than MVC.

The biggest challenge was wrapping my head around the architectural pattern of MVC and front-end web applications in general. Using ES6 modules to import and export components was very new to me at that point, but I took my time to slowly build up my understanding of the project. I played with the project, the data I received from the API, and did my research on the MVC architecture pattern.

Thankfully, I gave a detailed explanation of the MVC pattern for this project on the GitHub repository. This demonstrated that I understood what was being asked of me, and my instructor was impressed by what I had written. To this day, I still occasionally read my write-up for reference. Who else better to explain a concept to you than yourself!
