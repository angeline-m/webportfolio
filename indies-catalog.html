Project Experience

I was tasked with creating a full CRUD (create, read, update, delete) catalog from scratch using PHP and MySQL. The mandatory features were catalog filtering, login functionality, a random entry feature, and a search page. For speed and ease, I used Bootstrap 4.5 and FontAwesome for styling. For database management, I used phpMyAdmin to get the database started.

Users can view, sort, and filter through the catalog. They can view the individual page for each game. Once logged in, the user can update or delete any game entry and add new games as well. The cover image for the games, once uploaded and the entry is submitted, can be replaced afterwards.

I went above and beyond the requirements. I thought how a user would think and designed the website and added features according to my own experience on catalog sites. I decided to turn the search page into a global search bar for ease of use for the user. Having to go on a separate page to make a query worked against the natural flow of the site. I also thought it was wise to add sorting options if a user wants to use them. This is based on my own experience on online shopping sites, where I frequently use the sorting options to show the items I prefer to see first. The sorting feature can be used in conjunction with the filters or on its own. Lastly, I added an optional field to include a game trailer using the video code from a trailer of the game on YouTube. As a video editor, I believe videos are a great way to communicate information quickly and effectively to users.

The biggest challenge of this project was getting it up and running on the host provider. The version of PHP on the provider is not identical to the one I used, so extra time and research was put in to ensure the site could go live. I estimate that about 30% of the development time was due to hosting issues, which was unexpected
